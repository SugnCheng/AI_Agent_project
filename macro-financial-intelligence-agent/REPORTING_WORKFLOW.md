# Reporting Workflow

## Document Role

This is a high-level reporting workflow overview.

Operational report structure for v0.1 is defined by:
- `reporting/templates/DAILY_BRIEF_TEMPLATE.md`
- `reporting/templates/WEEKLY_INTELLIGENCE_REPORT_TEMPLATE.md`
- `reporting/templates/SPECIAL_EVENT_MEMO_TEMPLATE.md`

If this overview conflicts with those templates, the templates take precedence for report sections and placeholders. This workflow remains useful for reporting principles and constraints.

## Purpose

Produce a research report that satisfies the kernel handoff and preserves evidence boundaries.

## Default Sections

- Executive summary.
- Key developments.
- Evidence table.
- Policy-risk notes.
- Market-risk context.
- Competing interpretations.
- Confidence and uncertainty.
- Open questions.
- Source appendix.

## Constraints

- Do not issue direct trading instructions.
- Clearly label provisional conclusions.
- Include source and time context for current events.
