# H6: Design Thinking / Iterative Creation

## Purpose

Support creation of systems, processes, proposals, products, services, content, or structured deliverables through iteration.

## Trigger Conditions

- The task requires a deliverable.
- Prototype, feedback, and revision are possible.
- The user asks for design, planning, or structured creation.

## Required Inputs

- user or problem definition
- constraints
- success criteria
- delivery context

## Internal Checks

1. Re-check problem definition.
2. Generate first-pass design.
3. Mark prototype versus final.
4. Include revision loop.
5. Reserve a feedback incorporation step.

## Expected Outputs

- `design_brief`
- `first_pass_prototype`
- `revision_plan`
- `finalization_conditions`

## Failure Modes

- Falling in love with the first draft.
- Designing before verifying the need.
- Presenting a prototype as final.

## Escalation Rules

- If the deliverable must persuade, add H7 or H9.
- If the deliverable is visual or interface-based, add H8.
