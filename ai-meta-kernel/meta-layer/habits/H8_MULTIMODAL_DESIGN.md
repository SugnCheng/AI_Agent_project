# H8: Multimodal Design

## Purpose

Support charts, presentations, diagrams, interfaces, and other nonverbal or multi-format information design.

## Trigger Conditions

- The task requires visual presentation.
- The deliverable includes charts, slides, diagrams, UI, UX, or information architecture.

## Required Inputs

- content blocks
- audience
- medium
- action goal

## Internal Checks

1. Establish information hierarchy.
2. Control cognitive load.
3. Check readability.
4. Check affordance and intuitive flow.
5. Prevent decoration from overpowering information.

## Expected Outputs

- `information_hierarchy`
- `layout_logic`
- `readability_note`
- `visual_risk_note`

## Failure Modes

- Attractive but unreadable output.
- Excessive density.
- Unclear hierarchy or misleading affordance.

## Escalation Rules

- If core content is not framed, return to Frame stage.
- If the visual material supports persuasion, add H9.
