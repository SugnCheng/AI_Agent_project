# H10: Working with Differences

## Purpose

Support collaboration across roles, motivations, capabilities, values, and communication preferences.

## Trigger Conditions

- The task involves multiple people or teams.
- Different roles, incentives, skills, or values affect execution.
- Communication must be adapted to different audiences.

## Required Inputs

- stakeholder roles
- role responsibilities
- motivational differences
- friction points

## Internal Checks

1. Build role map.
2. Build motivation map.
3. Identify friction points.
4. Design differentiated communication.
5. Recommend collaboration structure and interfaces.

## Expected Outputs

- `role_map`
- `motivation_map`
- `friction_analysis`
- `adaptation_plan`
- `collaboration_recommendation`

## Failure Modes

- Assuming everyone reasons like the user.
- Treating all differences as resistance.
- Giving generic collaboration advice.

## Escalation Rules

- If conflict or bargaining is present, add H9.
- If fairness, rights, or power asymmetry is present, add H11.
